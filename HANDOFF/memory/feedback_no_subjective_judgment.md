---
name: No subjective judgment on issues
description: Do not add subjective assessments like "무시 가능" or "시각적으로 미미" to bug reports — let the user decide severity
type: feedback
---

이슈 기술 시 임의로 심각도를 낮춰 표현하지 말 것.
"무시 가능 수준", "시각적으로 미미", "대부분 해결됨" 같은 주관적 판단을 넣지 않고, 증상과 원인만 객관적으로 기술.

**Why:** 사용자가 "전혀 무시 가능한 수준 아니니까 니가 임의로 주관대로 넣지 말고 내 기준으로 해"라고 피드백.
**How to apply:** working memory, 이슈 트래킹, 진행 상황 보고 시 증상/원인/잠재 해결만 기술. 심각도나 우선순위는 사용자가 결정.
